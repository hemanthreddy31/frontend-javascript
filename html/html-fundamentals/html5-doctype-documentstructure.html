HTML5 DOCTYPE Declaration

The DOCTYPE declaration tells the browser which version of HTML the document is using. HTML5 simplified this to a clean, memorable declaration.

<!DOCTYPE html>
Why it matters: The DOCTYPE ensures the browser renders your page in standards mode, providing consistent behavior across different browsers and enabling modern HTML5 features.

Comparison with older DOCTYPEs:

<!-- HTML5 (current) --> <!DOCTYPE html> <!-- XHTML 1.0 Strict (legacy) --> <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd"> <!-- HTML 4.01 Transitional (legacy) --> <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">

The HTML Root Element

The <html> element is the root of every HTML document and should always include the lang attribute for accessibility and SEO.

<html lang="en"> <!-- All page content goes here --> </html>
Accessibility: The lang attribute helps screen readers pronounce content correctly and assists translation tools. Use specific language codes like "en-US" for American English or "fr-CA" for Canadian French when needed.

Language Code Examples:

  • lang="en" - English (general)
  • lang="en-US" - American English
  • lang="en-GB" - British English
  • lang="es" - Spanish
  • lang="fr" - French
  • lang="de" - German
  • lang="ja" - Japanese

The Head Section

The <head> contains metadata about the document that isn't displayed to users but is crucial for browsers, search engines, and accessibility tools.

Essential Head Elements:

<head> <!-- Character encoding (must be first) --> <meta charset="UTF-8"> <!-- Viewport for responsive design --> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <!-- IE compatibility --> <meta http-equiv="X-UA-Compatible" content="IE=edge"> <!-- Page title (appears in browser tab) --> <title>Page Title - Site Name</title> <!-- Meta description for SEO --> <meta name="description" content="Brief description of the page content"> <!-- Favicon --> <link rel="icon" type="image/x-icon" href="/favicon.ico"> </head>
Order matters: The charset declaration should be within the first 1024 bytes of the document. Place it first in the head section for best compatibility.

The Body Section

The <body> contains all visible content and should use semantic HTML5 elements to create a meaningful document structure.

Basic Body Structure:

<body> <!-- Skip link for accessibility --> <a href="#main-content" class="skip-link">Skip to main content</a> <!-- Site header --> <header role="banner"> <h1>Site Title</h1> <p>Site tagline or description</p> </header> <!-- Main navigation --> <nav role="navigation" aria-label="Main navigation"> <ul> <li><a href="#home">Home</a></li> <li><a href="#about">About</a></li> <li><a href="#contact">Contact</a></li> </ul> </nav> <!-- Main content --> <main id="main-content" role="main"> <h2>Page Heading</h2> <p>Main page content goes here...</p> </main> <!-- Site footer --> <footer role="contentinfo"> <p>© 2024 Site Name. All rights reserved.</p> </footer> </body>

Complete Semantic Document Structure

Here's a complete example showing proper HTML5 document structure with all essential elements:

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <meta http-equiv="X-UA-Compatible" content="IE=edge"> <title>Complete HTML5 Document Structure</title> <meta name="description" content="A complete example of proper HTML5 structure"> <meta name="author" content="Your Name"> <link rel="icon" type="image/x-icon" href="/favicon.ico"> <meta name="theme-color" content="#007bff"> </head> <body> <a href="#main-content" class="skip-link">Skip to main content</a> <header role="banner"> <h1>Website Title</h1> <p>Website description or tagline</p> </header> <nav role="navigation" aria-label="Main navigation"> <ul> <li><a href="#home" aria-current="page">Home</a></li> <li><a href="#about">About</a></li> <li><a href="#services">Services</a></li> <li><a href="#contact">Contact</a></li> </ul> </nav> <main id="main-content" role="main"> <section aria-labelledby="intro-heading"> <h2 id="intro-heading">Introduction</h2> <p>Welcome to our website...</p> </section> <section aria-labelledby="features-heading"> <h2 id="features-heading">Features</h2> <article> <h3>Feature One</h3> <p>Description of feature one...</p> </article> <article> <h3>Feature Two</h3> <p>Description of feature two...</p> </article> </section> </main> <aside role="complementary" aria-labelledby="sidebar-heading"> <h2 id="sidebar-heading">Related Information</h2> <p>Additional content related to the main content...</p> </aside> <footer role="contentinfo"> <p>© 2024 Website Name. All rights reserved.</p> <nav aria-label="Footer navigation"> <ul> <li><a href="#privacy">Privacy Policy</a></li> <li><a href="#terms">Terms of Service</a></li> </ul> </nav> </footer> </body> </html>
Key Accessibility Features:
  • Skip link for keyboard navigation
  • Proper heading hierarchy (h1 → h2 → h3)
  • ARIA labels and roles for screen readers
  • Language attribute on html element
  • Semantic elements for document structure
  • Current page indication with aria-current

Document Structure Best Practices

✅ Do:

  • Always include the HTML5 DOCTYPE declaration
  • Set the language attribute on the html element
  • Include charset declaration as the first meta tag
  • Use semantic HTML5 elements (header, nav, main, section, article, aside, footer)
  • Maintain proper heading hierarchy (h1 → h2 → h3, etc.)
  • Include viewport meta tag for responsive design
  • Add skip links for keyboard accessibility
  • Use ARIA labels and roles where appropriate

❌ Don't:

  • Skip the DOCTYPE declaration
  • Use multiple h1 elements on the same page
  • Skip heading levels (h2 directly to h4)
  • Forget the lang attribute on the html element
  • Use div elements when semantic elements are more appropriate
  • Place the charset meta tag after other content
  • Ignore accessibility features like skip links and ARIA labels
