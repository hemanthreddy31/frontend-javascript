Understanding HTML5 Semantic Elements

Semantic HTML elements provide meaning to your content, making it more accessible to screen readers, search engines, and other assistive technologies. This showcase demonstrates each semantic element with practical examples and accessibility considerations.

<section>

The Section Element

The <section> element represents a thematic grouping of content, typically with a heading. It's used to break up content into logical sections.

<section aria-labelledby="section-heading"> <h3 id="section-heading">Section Title</h3> <p>Section content goes here...</p> </section>
Accessibility: Sections should have headings and can use aria-labelledby to associate with their heading for better screen reader navigation.

Proper Heading Hierarchy

Semantic HTML requires a logical heading structure for accessibility and SEO:

<h1>Page Title (only one per page)</h1> <h2>Main Section</h2> <h3>Subsection</h3> <h4>Sub-subsection</h4> <h5>Minor heading</h5> <h6>Smallest heading</h6> <h2>Another Main Section</h2> <h3>Another Subsection</h3>
Accessibility: Screen readers use heading structure for navigation. Never skip heading levels (don't go from h2 to h4 without h3).
